## 2. Проблема

**Какую боль решаем:**

- Разрозненные инструменты: задачи, чаты, документация — в разных системах
- Долгий онбординг новых участников команды
- Отсутствие единого контекста между PO, аналитиками и разработчиками
- Сложность масштабирования команды без роста операционных затрат

**Целевая аудитория:**

- Product Owner и IT-лиды
- Команды 5–30 человек
- Пилотные проекты с AI-ассистентами

---

## 3. Решение — IT Team

**IT Team** — платформа управления мультиагентной IT-командой:

- Единое рабочее пространство: проекты, бэклог, спринты
- AI-агенты с ролями (BA, SA, Architect, Dev, QA и др.)
- Чаты с агентами через DeepSeek с системными промтами
- Общий чат проекта для всей команды

**Ценность для клиента:**

- Быстрее от идеи до технического описания
- Прозрачность статуса задач и спринтов
- Снижение рутины за счёт AI-агентов

---

## 4. Ключевые возможности

| Модуль | Описание |
|--------|----------|
| **Проекты и бэклог** | Kanban-доска, приоритеты, назначение агентов |
| **Спринты** | Планирование, привязка задач, статусы |
| **AI-агенты** | Настраиваемые роли и системные промты (только PO) |
| **Чаты** | Диалог с агентом, история, лимиты API |
| **Общий чат** | Сообщения видны всем участникам проекта |
| **Статистика** | Выполнение задач, активность агентов, тренды |

## 6. Безопасность и комплаенс

- **Аутентификация:** JWT, bcrypt для паролей
- **Изоляция данных:** задачи и проекты привязаны к владельцу (owner_id)
- **ПДн:** опция маскирования персональных данных в интерфейсе
- **Лимиты LLM:** 100 запросов в день на пользователя (настраивается)
- **Секреты:** API-ключи только в `.env` на сервере, не в git
- **HTTPS:** SSL на api.itteam.tech и itteam.tech

**Для enterprise-пилота уточнить:**

- [ ] Хостинг в РФ / on-premise
- [ ] Аудит логов
- [ ] SSO / LDAP
- [ ] Резервное копирование БД

## 7. План внедрения

| Этап | Срок | Действия |
|------|------|----------|
| **1. Пилот** | 1–2 недели | Регистрация, 1 проект, 3–5 агентов, обучение PO |
| **2. Настройка** | 1 неделя | Системные промты, спринты, роли в команде |
| **3. Оценка** | 1 неделя | Метрики из раздела «Статистика», обратная связь |
| **4. Масштабирование** | по согласованию | Доп. проекты, интеграции, SLA |

**Критерии успеха пилота:**

- [ ] Команда создаёт задачи и ведёт спринты в IT Team
- [ ] Агенты используются минимум 3 раза в неделю
- [ ] PO доволен скоростью подготовки артефактов
